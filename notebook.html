Abstract:¶

Financial forecasting remains challenging due to the volatile nature of the stock market. This tutorial demonstrates a machine learning pipeline using a PyTorch Long Short-Term Memory (LSTM) neural network to predict the S&P500 stock closing price. This problem uses tabular time-series data as the dataset, which makes an ideal case for this network. This tutorial will demonstrate how adjusting hyperparameters (in this tutorial, learning rate and hidden dimensions) affects predictive performance using Mean Squared Error as a metric. We can visualise performance using tables and graphs. Users can easily adjust hyperparameters and the chosen dataset.

After following this tutorial you will achieve the following learning objectives:

  • Preprocess time-series data suitable for a recurrent neural network.
  • Construct and Train an LSTM neural network using PyTorch libraries, utilising a GPU if possible.
  • Experiment with hyperparameters, especially learning rate, to optimise model performance.
  • Evaluate and visualise the impact of different configurations using Matplotlib.

How this tutorial differs: This tutorial will be easy to follow and trains an LSTM. We will train 3 different configurations and display their performance differences. We evaluate them using Mean Squared Error: $MSE=\frac{1}{n}\sum_{i=1}^{n}(Y_i-\hat{Y}_i)^2$

Step 5: Preprocess data¶

This step involves normalising the data using MinMaxScaler to scale the data from -1 to 1, helping the neural network converge faster. A sliding window of 30 days is used so the LSTM can see the last 30 days. The data is then split into 80% training and 20% for testing. Finally, the data in converted into PyTorch tensors. There are 1010 sequences to train on and 253 to test it on. This does not match up exactly with 365 days in a year because not every day is a trading day. The sequence size of 30 is seen, meaning 30 consecutive days are used to predict the next day. 1 indicates the amount of data recorded per day, and because we are only recording the closing price, this is the only feature.

###This code is adapted from the sequence generation function from the GeeksforGeeks (2023) tutorial

#Create sequences of data for time series (Sliding Window)
def create_sequences(data, seq_length):
    xs, ys = [], []
    for i in range(len(data)-seq_length-1):
        x = data[i:(i+seq_length)]
        y = data[i+seq_length]
        xs.append(x)
        ys.append(y)
    return np.array(xs), np.array(ys)

seq_length = 30 #Look back 30 days
X, y = create_sequences(scaled_data, seq_length)

Step 6: Define hyperparameters¶

This step consists of defining the hyperparameters.

  • Architectural hyperparameters determine how the model looks.
    • input_dim : how many features are inputted, being the closing price.
    • hidden_dim : the number of neurons inside the LSTM cell. This is kept as a control of 32 nodes.
    • output_dim : how many features are outputted, again being just the closing price.
    • num_layers : how many hidden layers there are, default to 1.
    • dropout : only used when there are 2 or more layers, otherwise it is ignored by the PyTorch library.
  • Training hyperparameters determine how the model learns.
    • epochs : the amount of loops, set at a constant of 200 to ensure convergence.
    • learning rate, lr : determines how extremely the network's weights are updated. A higher value can cause large "steps", possibly leading to more erratic jumps. Conversely a smaller learning rate.
  • Feel free to adjust these. In this tutorial, everything is kept as a control so you are able to see how the learning rate affects the performance.

#LSTM code from source RodolfoLSS (2025)
class LSTM(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_layers, output_dim, dropout):
        super(LSTM, self).__init__()
        self.hidden_dim = hidden_dim
        self.num_layers = num_layers
        
        #LSTM Layer
        self.lstm = nn.LSTM(input_dim, hidden_dim, num_layers, batch_first=True)
        #Fully Connected Layer for output
        self.fc = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        #Initialise hidden and cell states
        h0 = torch.zeros(self.num_layers, x.size(0), self.hidden_dim).to(x.device).requires_grad_()
        c0 = torch.zeros(self.num_layers, x.size(0), self.hidden_dim).to(x.device).requires_grad_()
        #Forward propagate LSTM
        out, (hn, cn) = self.lstm(x, (h0.detach(), c0.detach()))
        #Decode the hidden state of the last time step
        out = self.fc(out[:, -1, :]) 
        return out

def train_model(model, lr, epochs):
    criterion = nn.MSELoss()
    optimiser = torch.optim.Adam(model.parameters(), lr=lr)
    train_losses = []
    start_time = time.time()
    
    for epoch in range(epochs):
        model.train()
        optimiser.zero_grad()
        
        #Forward pass
        y_pred = model(X_train)
        loss = criterion(y_pred, y_train)
        
        #Backward pass and optimise
        loss.backward()
        optimiser.step()
        train_losses.append(loss.item())
        
    duration = time.time() - start_time
    
    #Calculate Test Loss
    model.eval()
    with torch.no_grad():
        test_pred = model(X_test)
        test_loss = criterion(test_pred, y_test).item()
    return model, train_losses, test_loss, duration, test_pred

Step 10: Evaluation¶

From the training loss curves, we can observe how the learning rate affects the behaviour during gradient descent.

  • The learning rate with the best performance is the baseline Config A
  • The higher learning rate, config B (Blue) takes very large steps. This is indicated by the large spikes. The model is bouncing back and forth.
  • The lower learning rate, config C (Green) is very smooth and find a slow, stable decrease.

The final test loss curves also help our understanding.

  • Config A has the lowest MSE
  • Config B has a worse MSE, showing that the large learning rate has prevented it from finding the most optimal weights because of the large jumps
  • Config C shows a smooth convergence but the learning rate is too small, meaning that it was stuck in a sub optimal local minimum compared to Config A.

For the hidden dimension curves, we can also interpret the behaviour.

  • The lower capacity model performs worse than the baseline because the model does not have enough parameters to learn the complex movement of the stock market.
  • The higher capacity model actually outperforms the baseline because of the increased model capacity, causing it to achieve the lowest Test MSE.

The predictive curve shows how the different models really perform on the test set. You can see how much each one varies from the actual results. The closest one being Config E. But each model roughly predicts peaks and drops well enough, following a similar enough pattern. Generally, the models understimate the magnitude of growth.

The act of hyperparameter tuning is a balancing act. If the network is too small or the learning rate is suboptimal, the model can underfit and struggle to learn the trends, If the network is too massive, it can start to overfit, which could be what is happening to Config E. You can adjust these as you need.
